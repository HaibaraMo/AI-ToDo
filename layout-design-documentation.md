# 界面布局设计方案说明文档

## 1. 问题背景

当前界面存在三个功能模块（待办事项、工时记录、周期事务提醒）并排排列导致的视觉拥挤问题。作为次一级功能，周期事务提醒的位置和展示方式需要重新设计以优化整体界面布局。

## 2. 设计目标

- 明确区分主功能与次功能的视觉层级
- 解决三个功能并排导致的拥挤问题
- 保持界面的简洁性和操作的便捷性
- 维持整体设计风格的一致性

## 3. 设计方案

### 方案1：标签页布局

#### 设计思路
- 将三个模块整合到标签页中
- 主标签（"主功能"）显示待办事项和工时记录（并排布局）
- 次标签（"周期提醒"）单独显示周期事务提醒模块（全屏宽度）
- 通过标签页切换来管理不同功能区域

#### 界面结构
```
┌─────────────────────────────────────────────────────────────┐
│  [主功能] [周期提醒]                                       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌───────────────────┐  ┌───────────────────┐               │
│  │  待办事项模块      │  │  工时记录模块      │               │
│  └───────────────────┘  └───────────────────┘               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 交互流程
1. 默认显示"主功能"标签，展示待办事项和工时记录模块
2. 点击"周期提醒"标签，切换到周期事务提醒模块
3. 点击"主功能"标签，返回主功能模块

#### 优缺点
- **优点**：
  - 完全分离主功能和次功能，视觉层级清晰
  - 每个标签页内的内容更加集中，减少视觉干扰
  - 周期提醒模块获得更大的展示空间
- **缺点**：
  - 需要额外的点击操作来访问周期提醒功能
  - 标签页切换可能打断用户的工作流

#### 适用场景
- 适用于功能使用频率差异较大的情况，周期提醒功能使用频率较低
- 适用于屏幕宽度有限的设备，需要更有效的空间利用
- 适用于希望主界面保持简洁的用户

### 方案2：垂直堆叠布局

#### 设计思路
- 垂直堆叠三个模块，待办事项和工时记录在上（并排布局）
- 周期事务提醒模块在下，并添加折叠/展开功能
- 通过折叠状态减少次功能对主功能的视觉干扰

#### 界面结构
```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  ┌───────────────────┐  ┌───────────────────┐               │
│  │  待办事项模块      │  │  工时记录模块      │               │
│  └───────────────────┘  └───────────────────┘               │
│                                                             │
│  ┌───────────────────────────────────────────────────────┐   │
│  │  [周期事务提醒] ▼                                    │   │
│  ├───────────────────────────────────────────────────────┤   │
│  │  周期事务提醒内容                                      │   │
│  └───────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 交互流程
1. 默认显示所有模块，周期事务提醒模块处于展开状态
2. 点击周期事务提醒模块的标题栏或折叠按钮，可折叠/展开该模块
3. 折叠状态下，仅显示模块标题，节省界面空间

#### 优缺点
- **优点**：
  - 所有功能都在同一页面中，无需标签页切换
  - 折叠功能可根据需要隐藏次功能，保持界面整洁
  - 操作路径更短，最多1次点击即可访问/隐藏周期提醒
- **缺点**：
  - 展开状态下仍然可能存在垂直方向的空间占用
  - 模块之间的视觉区分度不如标签页布局

#### 适用场景
- 适用于希望所有功能都在同一视野内的用户
- 适用于周期提醒功能使用频率中等的情况
- 适用于屏幕高度充足的设备

## 4. 技术实现

### 方案1：标签页布局
- **HTML结构**：使用`tab-container`、`tab-nav`、`tab-content`等类创建标签页结构
- **CSS样式**：实现标签页导航、内容切换的视觉效果
- **JavaScript**：实现标签页切换的交互逻辑

### 方案2：垂直堆叠布局
- **HTML结构**：使用`collapsible-module`、`module-header`、`module-content`等类创建可折叠模块
- **CSS样式**：实现模块折叠/展开的动画效果
- **JavaScript**：实现模块折叠/展开的交互逻辑

## 5. 响应式设计

### 方案1：标签页布局
- 在小屏幕设备上，主功能标签内的两个模块将垂直堆叠
- 标签页导航保持不变

### 方案2：垂直堆叠布局
- 在小屏幕设备上，所有模块都将垂直堆叠
- 折叠功能保持不变，继续节省空间

## 6. 选择建议

- **选择方案1**：如果您希望主界面更加简洁，周期提醒功能使用频率较低，或者屏幕宽度有限
- **选择方案2**：如果您希望所有功能都在同一页面中，周期提醒功能使用频率中等，或者屏幕高度充足

两种方案都保持了整体设计风格的一致性，并且都解决了原始布局的视觉拥挤问题。
